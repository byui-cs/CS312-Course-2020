Transformations¶

This week we will be learning how to move, scale, rotate and flip images using OpenCV and Numpy.

translate¶

Translation in computer vision and graphics means moving points and/or images on the screen or in an image. The easiest method of translating an image is to use numpy's features. The shape attribute must be used.

In [2]:
# Create a large image and translate the tricycle image to the center
# Note that the slicing values must equal the width and height of the tricycle image
new_image = np.zeros((400, 600, 3), dtype=np.uint8)

rows, cols, ch = image.shape
width_spacing = (400 - rows) // 2
height_spacing = (600 - cols) // 2

new_image[width_spacing:rows+width_spacing, height_spacing:cols+height_spacing] = image

plt.imshow(cv2.cvtColor(new_image, cv2.COLOR_BGR2RGB))
plt.show()

resizing¶

OpenCV has a resize() function that will allow you to resize an images you are working with. The following is from the OpenCV documentation.

cv2.resize(src, dsize[, dst[, fx[, fy[, interpolation]]]])

Parameters:

Parameter required Description
src [required] source/input image
dsize [required] desired size for the output image
fx [optional] scale factor along the horizontal axis
fy [optional] scale factor along the vertical axis
interpolation [optional] flag that takes one of the following methods. INTER_NEAREST – a nearest-neighbor interpolation INTER_LINEAR – a bilinear interpolation (used by default) INTER_AREA – resampling using pixel area relation. It may be a preferred method for image decimation, as it gives moire’-free results. But when the image is zoomed, it is similar to the INTER_NEAREST method. INTER_CUBIC – a bicubic interpolation over 4×4 pixel neighborhood INTER_LANCZOS4 – a Lanczos interpolation over 8×8 pixel neighborhood
- - -

Note that there are different options for interpolation. This is used to decide how pixels are removed from an image if it is shinking and increasing in size. Details on interpolation

flipping¶

Flipping an image is easy with the flip() function. Flip() has 3 arguments:

  • src – input array.
  • dst – output array of the same size and type as src.
  • flipCode – a flag to specify how to flip the array; 0 means flipping around the x-axis and positive value (for example, 1) means flipping around y-axis. Negative value (for example, -1) means flipping around both axes (see the discussion below for the formulas).

rotation (OpenCV Function)¶

Rotation of an image is very easy in OpenCV, however there are issues with losing parts of an image. See the examples below.

In [4]:
def rotateImage(image, angle, position=None):
    if (position == None):
        image_center = tuple(np.array(image.shape[1::-1]) / 2)
    else:
        image_center = position
    rot_mat = cv2.getRotationMatrix2D(image_center, angle, 1.0)
    result = cv2.warpAffine(image, rot_mat, image.shape[1::-1], flags=cv2.INTER_LINEAR)
    return result

Blending¶

OpenCV has a great function to blend two images together. It isn't the only method. The function is addWeighted().

The function is going to create a new image based on:

  • the two images
  • alpha a weighted value for the amount of image 1 to use in the new image
  • beta a weighted value for the amount of image 2 to use in the new image

Note: the two images must be the same size!

  • OpenCV documentation on addWeighted
  • addWeighted() Example
In [7]:
# Example of blending two images
# Notice that the final results is a blend of both images but half brightness

src1 = cv2.imread('blend.png', cv2.IMREAD_COLOR)
src2 = cv2.flip(src1, 1)

# add or blend the images
dst = cv2.addWeighted(src1, 0.5, src2, 0.5, 0.0)

# save the output image
print('Source image 1')
plt.imshow(cv2.cvtColor(src1, cv2.COLOR_BGR2RGB))
plt.show()

print('Source image 2')
plt.imshow(cv2.cvtColor(src2, cv2.COLOR_BGR2RGB))
plt.show()

print('Blended Image')
plt.imshow(cv2.cvtColor(dst, cv2.COLOR_BGR2RGB))
plt.show()
